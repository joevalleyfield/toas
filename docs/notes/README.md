# Notes

Short-form product and architecture notes that are not tasks.

## Purpose

Use this directory for ideas, observations, and operating heuristics that may or may not become roadmap work.

## File naming

Use one file per note:

- `YYYY-MM-DD-short-topic.md`

Examples:

- `2026-04-08-context-without-prefix-caching.md`
- `2026-04-09-frontier-failure-continuations.md`

## Suggested structure

Keep each note brief and practical:

- `Insight`
- `Why It Matters`
- `Implications`
- `Open Questions` (optional)

## Relationship To Tasks

- Tasks live in `tasks/` and represent intended or active work.
- Notes here are idea capture and design thinking.
- Promote a note into a task when it becomes implementation work.
