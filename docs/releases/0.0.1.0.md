# TOAS 0.0.1.0

Release date: 2026-07-06

## Summary

This release tags the first post-`0.0.0.0` history/projection slice as the
current stable checkpoint. The headline change is a coherent shift toward
hot-history defaults and explicit selected-source controls across graph,
history, heads, transcript, and llm-input surfaces.

## Highlights

- Added segmented event-index stitching and logical-history graph projection so
  graph/history queries can reason over hot plus selected sealed sources
  without lowering the hot-default operator path.
- Made `history`, `heads`, `transcript`, and `llm-input` default to hot
  lineage/history while adding explicit selected-source controls where needed.
- Added selected-source projection anchors and diagnostics so transcript- and
  llm-input-facing surfaces explain stitched provenance more clearly.
- Added graph source selection, local neighborhood selection, and stitched
  neighborhood alias handling to make segmented-history inspection more
  precise.
- Added a root-divergence salvage helper and documented history-recovery
  specimens for operators working through broken-history edge cases.

## Notable Fixes

- Fixed graph projection to operate over logical history instead of stitched
  physical ordering.
- Fixed logical-index lookup and source-qualified local-id semantics in
  selected-source history views.
- Fixed stitched history identity checks so scoped history queries reject
  ambiguous identity matches cleanly.
- Fixed runtime parent-root divergence handling by routing that state to an
  explicit sentinel parent shape.
- Promoted `ruff` and `mypy` into the routine `scripts/check.sh` gate so the
  release check now enforces lint and type cleanliness alongside tests.
