# TOAS 0.0.0.0

Release date: 2026-06-28

## Summary

This is the first TOAS stable checkpoint under the lightweight tagged-release
process. The release records the current local/operator-first system after the
LLM backend generalization work, acceptance replay revival, and release-gate
documentation landed.

## Highlights

- Added the Gemini REST backend path, including a zero-dependency
  `GeminiRESTDriver`, provider-aware request preparation, Gemini default base
  URL discovery, and LLM harness support for Gemini model listing.
- Refactored LLM transport behind an `LLMDriver` protocol while preserving the
  OpenAI-compatible path and startup behavior.
- Restored replay-only acceptance coverage to the routine check spine and
  documented the local `scripts/check.sh` release gate.
- Removed the standalone `toas rebuild` writeback surface in favor of explicit
  transcript projection/writeback through `toas transcript <head_id>`.
- Added `toas llm-input --envelope` so operators can inspect packet-inclusive
  model input shaping.
- Refined task/workboard/roadmap governance around active history-surface,
  release, and acceptance-test follow-up lanes.

## Notable Fixes

- Fixed Gemini request compatibility by blacklisting OpenAI-only `extra_body`
  parameters and mapping thinking configuration into Gemini native
  `thinkingConfig`.
- Preserved `llm_provider` while copying settings for request preparation.
- Pre-warmed LLM imports for host startup paths to avoid request event
  timeouts.
- Hardened test isolation against local `TOAS_LLM_*` environment variables.
- Improved workboard objective parsing for alternate task shapes.

## Verification

- `scripts/check.sh` passed on 2026-06-28 outside the sandbox:
  - default pytest: `2458 passed, 17 deselected`
  - coverage: `100.00%`
  - replay-only acceptance: `9 passed`

The same check was first attempted inside the Codex sandbox and failed on
environment permissions for local socket binds and Vim home-file writes; the
unsandboxed rerun passed.

## Deferred

- Package publication, distribution artifacts, signing, and support-window
  policy remain out of scope for the current release process.
- Release helper tooling remains tracked separately under
  `260627-release-helper-tooling`.
- Hosted CI remains deferred; the current gate is the documented local routine
  check set.
