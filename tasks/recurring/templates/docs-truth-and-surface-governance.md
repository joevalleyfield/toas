# Recurring: Docs Truth and Surface Governance

## Purpose
Keep roadmap/capabilities/protocol docs aligned with implemented reality and task-coupled directional content.

## Trigger
- Monthly
- Or after major architecture/runtime direction updates

## Inputs
- `docs/roadmap.md`
- `docs/capabilities.md`
- protocol/runtime direction notes
- open/closed tasks touched by doc claims

## Checklist
- Verify CURRENT vs DIRECTIONAL vs DRAFT labeling remains accurate
- Ensure directional/proposed content is linked to open tasks
- Ensure recently implemented content points to closure evidence
- Compress roadmap historical bloat; keep now/next/open arcs legible
- Check docs for contradictions across surfaces

## Output
- Dated run artifact
- Focused follow-on doc/code tasks when drift is discovered
