## Goal

Treat entrainment as a real prompt/runtime capability rather than an ad hoc rescue technique.

## Scope

- create prompt variants that teach the TOAS action protocol explicitly
- compare terse instructions against few-shot or demonstration-backed variants
- determine when entrainment is necessary to keep the backend inside the intended protocol

## Behavior

- prompt assets can express more than one protocol-teaching strategy
- TOAS has a clearer stance on when lightweight prompting is enough and when stronger setup is required

## Done When

- the prompt library includes at least one explicit protocol-entrainment path
- the repo documents what that path is meant to fix
